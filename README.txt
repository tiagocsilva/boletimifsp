- AS COORDENDAS DE CLIQUE DO BOT ESTÃO MAPEADAS PARA UMA RESOLUÇÃO DE TELA COM 1366x768

- PARA QUE O SISTEMA FUNCIONE CORRETAMENTE É NECESÁRIO QUE O NAVEGADOR PRINCIPAL SEMPRE ESTEJA NA TELA VISÍVEL E QUE O
O MOUSE POSSA SER MOVIDO MESMO QUANDO A TODO O TEMPO, ISSO IMPEDE QUE O SERVIÇO POSSA SER INICIADO PELO BASH REMOTAMENTE
OU ATÉ MESMO QUE O SERVIÇO RODE ATRAVÉS DE UM SERVIÇO DO WINDOWS, PARA OS TESTES FOI UTILIZADO UM SERVIDOR COM UBUNTU
SERVER INSTALADO.

- PARA QUE O SISTEMA FUNCIONE É PRECISO QUE NO SERVIDOR QUE HOSPEDARÁ A API ESTEJA INSTALADO O "FFMPEG"

- É INTERESSANTE, MAS NÃO MANDATÓRIO, UTILIZAR UM SERVIÇO DE DNS, COMO POR EXEMPLO O "NO-IP", POIS EM ALGUNS CASOS
O GOOGLE PODE BLOQUEAR O IP PÚBLICO DO SERVIDOR PARA A RESOLUÇÃO DOS CAPTCHAs SENDO NECESSÁRIO A TROCA DO MESMO, NESTE
CASO SE ESTIVER USANDO UM DNS, NÃO TERÁ PROBLEMA ALGUM COM O APLICATIVO, EM OUTROS CASOS SERIA NECESSÁRO UMA NOVA BUILD
PARA REAPONDAR O APP PARA O NOVO IP.
